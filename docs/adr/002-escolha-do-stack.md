# ADR-002 — Escolha do Stack Tecnológico
**Data:** 03/11/2025  
**Status:** Implementado  

## 1. Contexto  
Era necessário escolher tecnologias modernas e performáticas para lidar com grande volume de dados e proporcionar interatividade.

## 2. Decisão  
- **Backend:** Python + FastAPI + SQLAlchemy + PostgreSQL  
- **Frontend:** Nuxt.js (Vue.js) + Tailwind CSS + Chart.js  

## 3. Alternativas Consideradas  
- **Django:** muito pesado.  
- **Flask:** exigia mais configuração manual.  
- **React/Angular:** curva de aprendizado maior.

## 4. Justificativas  
- **FastAPI:** rápido e com tipagem forte.  
- **SQLAlchemy:** seguro e eficiente.  
- **Nuxt.js:** reativo e organizado.  
- **Tailwind CSS:** estilo ágil e moderno.  
- **Chart.js:** biblioteca popular e flexível.

## 5. Consequências  
- ✅ Stack moderno e produtivo.  
- ✅ Curva de aprendizado baixa.  
- ❌ Múltiplas dependências externas.
